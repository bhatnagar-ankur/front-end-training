HTML Images Training: Mastering the img Tag

Welcome to HTML images fundamentals! This comprehensive guide covers everything you need to know about adding and managing images in HTML.

1. Adding Images with img Tag

Basic Image Syntax

<img src="image-path" alt="description">

This is an actual image using the img tag

Complete Image Examples

<!-- Local image -->
<img src="images/logo.png" alt="Company Logo">

<!-- Image from subdirectory -->
<img src="assets/photos/sunset.jpg" alt="Beautiful sunset over mountains">

<!-- External image -->
<img src="https://example.com/images/banner.jpg" alt="Website banner image">

Self-Closing Tag

Important: The <img> tag is a void element (self-closing). It doesn't have a closing tag like </img>.

  • HTML5: <img src="photo.jpg" alt="Photo">
  • XHTML: <img src="photo.jpg" alt="Photo" />

2. Image Attributes: src, alt, width, height

src Attribute (Source)

The src attribute specifies the path to the image file.

<!-- Relative paths -->
<img src="photo.jpg" alt="Photo in same directory">
<img src="images/photo.jpg" alt="Photo in images folder">
<img src="../images/photo.jpg" alt="Photo in parent directory">

<!-- Absolute paths -->
<img src="/assets/images/photo.jpg" alt="Photo from root directory">
<img src="https://example.com/photo.jpg" alt="External photo">

alt Attribute (Alternative Text)

The alt attribute provides alternative text for screen readers and when images fail to load. 

<!-- Descriptive alt text -->
<img src="dog.jpg" alt="Golden retriever wearing business tie at office desk">

<!-- Functional alt text -->
<img src="search-icon.png" alt="Search">

<!-- Empty alt for decorative images -->
<img src="decoration.png" alt="">
Decorative element (empty alt)

width and height Attributes

These attributes set the display dimensions of the image in pixels.

<!-- Setting specific dimensions -->
<img src="photo.jpg" alt="Professional worker relaxing at desk" width="300" height="200">

<!-- Setting only width (maintains aspect ratio) -->
<img src="photo.jpg" alt="Team meeting with laughter" width="300">

<!-- Setting only height (maintains aspect ratio) -->
<img src="photo.jpg" alt="Creative photography setup" height="200">

<!-- Large banner image -->
<img src="banner.jpg" alt="Business analytics dashboard banner" width="800" height="200">

Combined Attributes Example

<img src="products/laptop.jpg" 
     alt="Professional worker with laptop showing frustrated expression with hammer" 
     width="400" 
     height="300">
Best Practice: Always include the alt attribute for accessibility. Use width and height to prevent layout shifts during loading.

3. Best Practices for Web Images

File Formats and When to Use Them

Format Best For Features File Size
JPEG (.jpg) Photographs, complex images Lossy compression, no transparency Small to medium
PNG (.png) Graphics with transparency, logos Lossless compression, transparency Medium to large
WebP (.webp) Modern web images Better compression than JPEG/PNG Small
SVG (.svg) Icons, simple graphics Vector-based, scalable Very small
GIF (.gif) Simple animations Limited colors, animation support Small to medium

Image Optimization Guidelines

<!-- Optimized image with proper attributes -->
<img src="optimized-photo.webp" 
     alt="Creative product photography with pink backdrop and kettle"
     width="600" 
     height="400"
     loading="lazy">

<!-- Responsive image with srcset -->
<img src="photo-medium.jpg" 
     srcset="photo-small.jpg 480w, 
             photo-medium.jpg 800w, 
             photo-large.jpg 1200w"
     sizes="(max-width: 480px) 480px, 
            (max-width: 800px) 800px, 
            1200px"
     alt="Professional photography studio setup with multiple lights"
     width="800" 
     height="600">

Writing Effective Alt Text

Good Alt Text Examples

<!-- Descriptive and specific -->
<img src="chart.png" alt="Sales increased by 25% from January to March 2024">

<!-- Functional description -->
<img src="download-icon.svg" alt="Download PDF report">

<!-- Context-appropriate -->
<img src="author.jpg" alt="Dr. Sarah Johnson, lead researcher">

Poor Alt Text Examples

<!-- Too generic -->
<img src="chart.png" alt="chart">

<!-- Redundant information -->
<img src="photo.jpg" alt="Photo of a person">

<!-- Missing context -->
<img src="icon.png" alt="icon">

Performance Best Practices

  • Use appropriate file formats based on image content
  • Optimize file sizes without compromising quality
  • Include width and height to prevent layout shifts
  • Use lazy loading for images below the fold: loading="lazy"
  • Implement responsive images with srcset and sizes
  • Use WebP format when possible for better compression
Accessibility Warning: Never use alt="" for important images. Only use empty alt text for purely decorative images that add no informational value.

4. Figure and Figcaption Elements

Basic Figure Structure

<figure>
    <img src="diagram.png" alt="Professional photography studio with equipment setup">
    <figcaption>Figure 1: Complete professional photography workflow</figcaption>
</figure>

Figure with Multiple Images

<figure>
    <img src="before.jpg" alt="Office worker stressed with laptop">
    <img src="after.jpg" alt="Same worker relaxed and smiling">
    <figcaption>Before and after: The power of good coffee in the workplace</figcaption>
</figure>

Figure with Code Example

<figure>
    <pre><code>
function greetUser(name) {
    return `Hello, ${name}!`;
}
    </code></pre>
    <figcaption>Listing 1: JavaScript function for user greeting</figcaption>
</figure>

Figure with Table

<figure>
    <table>
        <thead>
            <tr>
                <th>Browser</th>
                <th>Market Share</th>
            </tr>
        </thead>
        <tbody>
            <tr>
                <td>Chrome</td>
                <td>65.1%</td>
            </tr>
            <tr>
                <td>Safari</td>
                <td>18.8%</td>
            </tr>
        </tbody>
    </table>
    <figcaption>Table 1: Browser market share data for 2024</figcaption>
</figure>

Figcaption Positioning

<!-- Caption at the top -->
<figure>
    <figcaption>Creative product photography with amusing props</figcaption>
    <img src="architecture.svg" alt="Creative kettle pouring pink liquid into stacked glasses">
</figure>

<!-- Caption at the bottom (more common) -->
<figure>
    <img src="results.png" alt="Happy business team collaborating around laptop">
    <figcaption>Results from team building exercise: 95% fun factor achieved</figcaption>
</figure>

When to Use Figure and Figcaption

  • Images that need captions or detailed descriptions
  • Diagrams and charts that require explanation
  • Code examples that need context or numbering
  • Tables and data that need titles or explanations
  • Any content that is referenced from the main text
Figure Best Practices:
  • Use <figure> for self-contained content that can be moved without losing meaning
  • Keep <figcaption> concise but descriptive
  • <figcaption> should be the first or last child of <figure>
  • Only use one <figcaption> per <figure>

5. Advanced Image Techniques

Responsive Images

<!-- Using picture element for art direction -->
<picture>
    <source media="(max-width: 480px)" srcset="hero-mobile.jpg">
    <source media="(max-width: 768px)" srcset="hero-tablet.jpg">
    <img src="hero-desktop.jpg" alt="Professional worker in modern office environment">
</picture>

<!-- Using srcset for different resolutions -->
<img src="photo.jpg" 
     srcset="photo-1x.jpg 1x, photo-2x.jpg 2x, photo-3x.jpg 3x"
     alt="High-resolution business team meeting photo">

Image Gallery with Figures

<div class="gallery">
    <figure>
        <img src="gallery1.jpg" alt="Dog in business attire at office desk">
        <figcaption>Employee of the Month</figcaption>
    </figure>
    
    <figure>
        <img src="gallery2.jpg" alt="Creative photography studio setup">
        <figcaption>Behind the scenes magic</figcaption>
    </figure>
    
    <figure>
        <img src="gallery3.jpg" alt="Professional worker with laptop in bright office">
        <figcaption>Modern workspace vibes</figcaption>
    </figure>
</div>
